# Codebase Knowledge

Last updated: 2026-03-07 (after task: branch-preflight-check, retrospective)

## Architecture Overview

This repo is a set of markdown prompt templates and configuration files consumed by AI coding agents. There is no executable code. The system guides agents through a 9-phase development workflow via prompt templates in `prompts/`, with configuration defaults in `defaults/`, language-specific review checklists in `review-profiles/`, and documentation in `docs/`.

Consuming projects reference `skill.md` from their agent config. Per-project overrides live in a config directory (typically `.task-system/`) in the consuming project. Task output (definition, plan, implementation tracking, etc.) is written to a tasks directory in the consuming project.

## Key Modules

### `skill.md`
- **Location**: `/skill.md`
- **Purpose**: Entry point for AI agents. Defines commands, phase flow, phase routing table, phase transitions, file resolution rules, backlog behavior, and cross-session behavior.
- **Key patterns**: File resolution uses priority order (project config → defaults) with per-file merge strategies (append or replace). Phase transitions are deterministic with skip conditions and loops.
- **Last verified**: 2026-03-07

### `prompts/`
- **Location**: `/prompts/`
- **Purpose**: Phase-specific prompt templates (00-setup through 09-retrospective). Each file contains complete instructions for one phase.
- **Key patterns**: Consistent structure: title → prerequisites → instructions → project-specific steps injection → output template → user review → state update → next steps. Phases reference each other's output files by name (e.g., Phase 5 reads `03-plan.md`). `05-implement.md` includes a 5-step preflight check sequence: (1) git working tree, (2) branch check with configurable `Branch Prefix` from `config.md`, (3) existing tests, (4) build, (5) record rollback point. The branch check detects `main`/`master`, suggests a feature branch name using issue identifiers and task slug, and handles stash/pop for uncommitted changes.
- **Last verified**: 2026-03-07

### `defaults/`
- **Location**: `/defaults/`
- **Purpose**: Baseline configuration files applied to all projects unless overridden.
- **Key patterns**: Four files with two merge strategies:
  - `coding-guidelines.md` — **append** (project adds rules to defaults)
  - `commit-message-format.md` — **replace** (project fully overrides)
  - `project-commands.md` — **replace** (project fully overrides)
  - `issue-trackers.md` — **replace** (project fully overrides). Configures
    issue tracker integration (Linear, Gitea) with env var names for
    credentials and non-secret settings like Gitea base URL and default repo.
  Files are self-documenting templates with comments explaining usage.
- **Last verified**: 2026-03-07

### `review-profiles/`
- **Location**: `/review-profiles/`
- **Purpose**: Language-specific code review checklists used in Phase 6 (Code Review) and Phase 8 (Final Polish).
- **Key patterns**: **Append** semantics — project profiles add checks to defaults. Auto-detected by language during Phase 2 research.
- **Last verified**: 2026-03-07

### `docs/`
- **Location**: `/docs/`
- **Purpose**: Detailed documentation for configuration, phases, customization, installation, troubleshooting, knowledge base, and git-excluded artifacts.
- **Key patterns**: Cross-referenced markdown files. `configuration.md` documents all config files with merge behavior summary table.
- **Last verified**: 2026-03-07

## Conventions & Patterns

- **File resolution**: Defined in `skill.md`. Each config file type has a heading describing resolution order and merge strategy.
- **Phase output files**: Named `NN-name.md` matching phase numbers. Downstream phases reference these by exact filename.
- **Phase transitions**: Defined in `skill.md` routing table. Each prompt file ends with "Next Steps" directing to the next phase.
- **Project-specific extensions**: `phase-steps/NN-phase-name.md` files in project config inject additional steps into phases (append semantics, no replacement).
- **State tracking**: `task-state.md` in each task directory tracks current phase, complexity, language(s), and phase history with timestamps.

## Infrastructure & Configuration

- **No build system** — pure markdown, no executable code.
- **Git**: Standard git repo. Task artifacts can be committed or excluded (controlled by `config.md` `artifact-tracking` field).
- **Repo-local pointer**: `.git/info/task-system` stores per-repo config/tasks paths (never committed).
